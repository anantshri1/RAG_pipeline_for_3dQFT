# RAG Pipeline for 3d Mirror Symmetry

A retrieval-augmented generation (RAG) system built over a theoretical physics PhD thesis on 3d $\mathcal{N}\leq2$ mirror symmetry. Vanilla LLMs hallucinate on this material ‚ÄĒ the subfield (planar Abelianization was introduced as a paradigm for treating these theories towards the end of 2024, which was the cutoff date for training of most LLMs) is largely absent from standard training data. This project builds and evaluates a RAG pipeline that grounds generation in the actual thesis content. 
> Note: the thesis itself is not available now, but the content of the relevant papers can be found [here](https://inspirehep.net/authors/2931640).

---

## Motivation

Standard LLMs conflate 3d mirror symmetry with its 2d (Calabi-Yau mirror pairs). The correct definition ‚ÄĒ an infrared duality exchanging **topological and flavour symmetries**, or equivalently order and disorder operators ‚ÄĒ requires retrieval from domain-specific literature. This makes the thesis an ideal RAG corpus: the content is real, the failure modes of naive LLMs are known, and the author can evaluate outputs as a domain expert.

---

## Corpus

- **Source:** PhD thesis "The ABCD's of Mirror Symmetry" (344 pages, LaTeX-compiled PDF)
- **Parser:** PyMuPDF ‚ÄĒ handles equations cleanly; TikZ quiver diagrams are lost but defined in surrounding prose
- **Clean corpus:** Pages 39‚Äď327 (289 pages) ‚ÄĒ front matter and bibliography excluded
- **Rationale for exclusion:** Bibliography pages produced hallucinated summaries (e.g. holomorphic 3-forms, Calabi-Yau geometry) in preliminary runs, adding noise without signal

---

## Architecture

### Notebook 1: Vanilla RAG (`RAG_Pipeline_3dMirrors.ipynb`)

- **Embedding model:** `BAAI/bge-large-en-v1.5` (top MTEB leaderboard, free, runs on T4)
- **Index:** `VectorStoreIndex` over 344 page-level documents
- **Retrieval:** Cosine similarity, top-k=5
- **Reader LLM:** Mistral-7B-Instruct-v0.3, 4-bit quantised via bitsandbytes

### Notebook 2: Hierarchical RAG (`RAG_Pipeline_3dMirrors_Hierarchical.ipynb`)

Two-level hierarchy per page:
- **Child node:** One-sentence summary generated by `claude-sonnet-4-5` ‚ÄĒ sharp retrieval signal
- **Parent node:** Full raw page text ‚ÄĒ rich generation context
- **Retrieval:** Cosine similarity over child summaries via `AutoMergingRetriever`; retrieved children promote to parents before generation
- **Why Claude for summaries:** Groq hallucinated domain-irrelevant concepts (holomorphic 3-forms, Calabi-Yau geometry); Claude produced physically accurate, terminology-preserving summaries, although Claude hallucinated "I don't see a passage" for section header pages, which must be removed in future runs.

**Key design decision:** Retrieval matches on distilled vocabulary; generation uses full page context. This separates the retrieval signal problem from the generation context problem.

## Failure Mode Analysis

**Q2 (operator map, page 315):** Vocabulary collision ‚ÄĒ multiple pages discuss operator mappings in mirror symmetry, causing the retrieval signal to diffuse. Potential fix: query decomposition or metadata filtering to Appendix B.

**Q4 (topological symmetry enhancement, page 130):** Vocabulary collision ‚ÄĒ multiple pages discuss symmetry enhancement in mirror symmetry, causing the retrieval signal to diffuse. Potential fix: query decomposition or metadata filtering.

**Q6 (singlet count, Appendix B):** Calculation-buried answer ‚ÄĒ the result (9 singlets) is implicit in a multi-step appendix calculation. No embedding-based retrieval strategy recovers this without chunk-level summarisation that explicitly states the final result. Fix: structured knowledge extraction (GraphRAG).

**Q8 (Marginally chiral theories, page 192):** Likely parsing issue ‚ÄĒ these theories are discussed on a graphics-heavy page.

**Q10 (2d limits, page 304):** Ranking problem ‚ÄĒ page 304 appears at position 7 in top-k retrieval, just below the cutoff for hierarchical RAG. 

**Q11 (Class S footnote, page 313):** Retrieval succeeded in hierarchical RAG (page 313 in top 5) but generation failed ‚ÄĒ the answer is in a footnote, which Mistral-7B likely ignored in favour of main text.

>Note: We see that HierarchicalRAG scores 8/11 on retrieval, but 6/11 on generation.
>Retrieval evaluation note: chunk previews (first 200 characters) were used for initial scoring but final retrieval scores reflect inspection of full parent node content. `AutoMergingRetriever` promotes child summary matches to full parent pages ‚ÄĒ answers may be present in the latter half of a page and not visible in truncated previews.

## Known Limitations

- **Quiver diagrams:** TikZ-compiled diagrams are not recovered by PyMuPDF. The system reasons from figure captions and surrounding prose only. Quiver topology is not directly retrievable.
- **Calculation-buried answers:** Results implicit in multi-step derivations are not retrievable without explicit summarisation of the final result.
- **LLM generation:** Mistral-7B-Instruct 4-bit runs close to T4 memory limits (14.5GB), causing OOM errors when combined with the embedding model. Generation evaluation (faithfulness, answer relevancy) is deferred to the reranking + LlamaParse iteration.

---

## Reranking Experiment with HierarchicalRAG
A cross-encoder reranker (`BAAI/bge-reranker-large`) was tested as a post-retrieval step, rescoring top-10 retrieved chunks before passing them to the reader. Unlike embedding-based retrieval which compares query and document vectors independently, a cross-encoder reads the query and each candidate chunk jointly, producing a more precise relevance score. This improved generation quality from 6/11 to 7/11, recovering the benefit of broader top-10 retrieval while keeping the context window tight. The reranker effectively acts as a precision filter on a wider candidate set ‚ÄĒ retrieve broadly, then rerank precisely.

## Future Work
- **LlamaParse:** Vision-capable PDF parser that recovers compiled TikZ output as figure descriptions; enables quiver topology to enter the retrieval index
- **RAGAS evaluation:** Full pipeline evaluation (faithfulness, answer relevancy, context precision, context recall) once generation is stable
- **GraphRAG:** Knowledge graph where nodes are theories/quivers and edges are dualities, RG flows, and quiver subtractions. Multi-hop questions (e.g. "what is the Coulomb branch of the mirror of USp(4) with 6 fundamentals?") require graph traversal, not chunk similarity. This is the natural architecture for a field whose mathematical structure is itself a graph.
- Swapping **Mistral-7B-Instruct 4-bit** for a stronger reader may improve the score.
