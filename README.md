# RAG Pipeline for 3d Mirror Symmetry

A retrieval-augmented generation system built over a theoretical physics PhD thesis on 3d $\mathcal{N}\leq2$ mirror symmetry. Vanilla LLMs hallucinate on this material — the subfield (USp quiver gauge theories, planar Abelian dualities, s-confinement, quiver subtraction) is largely absent from standard training data. This project builds and evaluates a RAG pipeline that grounds generation in the actual thesis content.

---

## Motivation

Standard LLMs conflate 3d mirror symmetry with its 2d/4d cousins (Calabi-Yau mirror pairs, Seiberg-Witten geometry). The correct definition — an infrared duality exchanging **topological and flavour symmetries**, or equivalently order and disorder operators — requires retrieval from domain-specific literature. This makes the thesis an ideal RAG corpus: the content is real, the failure modes of naive LLMs are known, and the author can evaluate outputs as a domain expert.

---

## Corpus

- **Source:** PhD thesis "The ABCD's of Mirror Symmetry" (344 pages, LaTeX-compiled PDF)
- **Parser:** PyMuPDF — handles equations cleanly; TikZ quiver diagrams are lost but defined in surrounding prose
- **Clean corpus:** Pages 39–327 (289 pages) — front matter and bibliography excluded
- **Rationale for exclusion:** Bibliography pages produced hallucinated summaries (e.g. holomorphic 3-forms, Calabi-Yau geometry) in preliminary runs, adding noise without signal

---

## Architecture

### Notebook 1: Vanilla RAG (`RAG_Pipeline_3dMirrors.ipynb`)

- **Embedding model:** `BAAI/bge-large-en-v1.5` (top MTEB leaderboard, free, runs on T4)
- **Index:** `VectorStoreIndex` over 344 page-level documents
- **Retrieval:** Cosine similarity, top-k=5
- **Reader LLM:** Mistral-7B-Instruct-v0.3, 4-bit quantised via bitsandbytes

### Notebook 2: Hierarchical RAG (`RAG_Pipeline_3dMirrors_Hierarchical.ipynb`)

Two-level hierarchy per page:
- **Child node:** One-sentence summary generated by `claude-sonnet-4-5` — sharp retrieval signal
- **Parent node:** Full raw page text — rich generation context
- **Retrieval:** Cosine similarity over child summaries via `AutoMergingRetriever`; retrieved children promote to parents before generation
- **Why Claude for summaries:** Groq hallucinated domain-irrelevant concepts (holomorphic 3-forms, Calabi-Yau geometry); Claude produced physically accurate, terminology-preserving summaries

**Key design decision:** Retrieval matches on distilled vocabulary; generation uses full page context. This separates the retrieval signal problem from the generation context problem.

## Failure Mode Analysis

**Q2 (operator map, page 315):** Vocabulary collision — multiple pages discuss operator mappings in mirror symmetry, causing the retrieval signal to diffuse. The correct page scores below the noise floor of thematically similar pages. Fix: query decomposition or metadata filtering to Appendix B.

**Q6 (singlet count, Appendix B):** Calculation-buried answer — the result (9 singlets) is implicit in a multi-step appendix calculation, never stated as a clean retrievable sentence. No embedding-based retrieval strategy recovers this without chunk-level summarisation that explicitly states the final result. Fix: structured knowledge extraction (GraphRAG).

**Q10 (2d limits, page 304):** Ranking problem — page 304 likely appears at position 6–8 in top-k retrieval, just below the cutoff. Fix: cross-encoder reranking (`BAAI/bge-reranker-large`).

---

## Known Limitations

- **Quiver diagrams:** TikZ-compiled diagrams are not recovered by PyMuPDF. The system reasons from figure captions and surrounding prose only. Quiver topology is not directly retrievable.
- **Calculation-buried answers:** Results implicit in multi-step derivations are not retrievable without explicit summarisation of the final result.
- **LLM generation:** Mistral-7B-Instruct 4-bit runs close to T4 memory limits (14.5GB), causing OOM errors when combined with the embedding model. Generation evaluation (faithfulness, answer relevancy) is deferred to the reranking + LlamaParse iteration.

---

## Future Work

- **Cross-encoder reranking** (`BAAI/bge-reranker-large`): expected to fix Q10 and improve borderline retrievals
- **LlamaParse:** Vision-capable PDF parser that recovers compiled TikZ output as figure descriptions; enables quiver topology to enter the retrieval index
- **RAGAS evaluation:** Full pipeline evaluation (faithfulness, answer relevancy, context precision, context recall) once generation is stable
- **GraphRAG:** Knowledge graph where nodes are theories/quivers and edges are dualities, RG flows, and quiver subtractions. Multi-hop questions (e.g. "what is the Coulomb branch of the mirror of USp(4) with 6 fundamentals?") require graph traversal, not chunk similarity. This is the natural architecture for a field whose mathematical structure is itself a graph.
